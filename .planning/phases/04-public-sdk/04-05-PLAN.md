<task type="auto">
  <name>Task 2: Add OIDC provenance publish workflow</name>
  <files>.github/workflows/publish.yml</files>
  <action>
    Create `.github/workflows/publish.yml`:
    - Trigger: `on: push: branches: [main]` (release flow) — the changesets Action opens a
      "Version Packages" PR; merging it triggers the publish.
    - Top-level `permissions:` set `contents: write`, `pull-requests: write`, `id-token: write`
      (id-token:write is REQUIRED for OIDC/provenance).
    - Single `release` job, `runs-on: ubuntu-latest`, `needs`-equivalent gating: ensure the build +
      SDK validation run first. Steps:
      checkout@v4 → pnpm/action-setup@v4 (v9) →
      setup-node@v4 with node 22, cache pnpm, AND `registry-url: https://registry.npmjs.org`
      (registry-url is what makes OIDC publishing work) →
      `pnpm install --frozen-lockfile` →
      `pnpm build` →
      `pnpm --filter @cyclops/core run lint:publish` (re-run the SDK gate as a pre-publish guard) →
      `changesets/action@v1` with `publish: pnpm release` (which runs `changeset publish`).
      Ensure npm provenance is enabled (npm >=9.5 auto-enables provenance when id-token:write +
      registry-url are present; add `NPM_CONFIG_PROVENANCE: true` env to be explicit).
    - Do NOT reference NPM_TOKEN anywhere (trusted publishing replaces it).
  </action>
  <verify>
    `python3 -c "import yaml; yaml.safe_load(open('.github/workflows/publish.yml'))"` parses clean.
    `grep -n 'id-token: write\|registry.npmjs.org\|changeset publish\|changesets/action\|lint:publish' .github/workflows/publish.yml` shows OIDC perms, registry, gate, and publish step.
    `grep -c 'NPM_TOKEN' .github/workflows/publish.yml` returns 0.
  </verify>
  <done>publish.yml publishes @cyclops/core via changesets + npm OIDC trusted publishing with provenance, re-running the SDK validation gate first, with no NPM_TOKEN.</done>
