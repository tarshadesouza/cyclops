<objective>
Establish the public SDK type surface: move crypto out of `packages/core` into a new private `@cyclops/internal` package, then add the `IDetector` interface and `DetectorContext` type to `packages/core`.

Purpose: `@cyclops/core` must be I/O-free (SDK-03) and expose the public detector contract (SDK-02). Crypto uses `node:crypto` and `process.env` — it cannot live in a publishable, I/O-free SDK. Consumers must be able to `implements IDetector` and pass a `DetectorContext`.
Output: `@cyclops/core` exports `IDetector`, `DetectorContext`, `DetectorResult` (and existing shared types) with no crypto; `@cyclops/internal` holds crypto; both apps compile against the new imports; detector files unchanged via alias.
</objective>
